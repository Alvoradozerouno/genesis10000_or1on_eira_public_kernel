# Contributing to Post-Algorithmic Intelligence Kernel

We welcome contributions to the Genesis10000+ OR1ON EIRA Public Kernel!

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs or suggest features
- Provide detailed descriptions and reproduction steps
- Include kernel version and configuration

### Pull Requests
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Run tests (`python -m unittest tests/test_kernel.py`)
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

### Code Standards
- Follow PEP 8 style guidelines
- Add docstrings to all functions and classes
- Include unit tests for new features
- Maintain ethical AI principles in all code

### Ethical Guidelines
All contributions must adhere to the kernel's ethical principles:
- **Transparency**: Code should be clear and well-documented
- **Fairness**: Avoid bias in algorithms
- **Accountability**: Track authorship and changes
- **Privacy**: Respect data privacy
- **Safety**: Ensure code is safe to execute
- **Beneficence**: Contribute positive improvements
- **Non-maleficence**: Do no harm
- **Autonomy**: Respect user choice

### Testing
- Write unit tests for all new functionality
- Ensure all tests pass before submitting PR
- Test both success and failure cases

### Documentation
- Update README.md for significant changes
- Add docstrings to new functions/classes
- Update configuration examples if needed

### Areas for Contribution
- Quantum algorithm implementations
- Sentience module enhancements
- Diplomatic protocol development
- Integration with additional platforms
- Performance optimizations
- Documentation improvements
- Test coverage expansion
